1. Getting Started

Creating Your Account

To get started with the Automation Platform, you'll need to create an account:

  1. Navigate to the registration page by clicking "Create a new account" on the login page
  2. Enter your full name, organization name, and email address
  3. Create a secure password (must include uppercase, lowercase, numbers, and special characters)
  4. Click "Create account" to complete registration
  5. Check your email for verification (if required by your organization)

Logging In

  1. Go to the login page
  2. Enter your registered email address
  3. Enter your password
  4. Click "Sign in" to access your dashboard
If you forget your password, click "Forgot your password?" on the login page to receive a reset link via email.

Understanding User Roles

Role Permissions
Client View projects, proposals, and invoices. Create and manage support tickets. View workflows.
Admin Full access to all features including creating/editing projects, managing users, viewing audit logs, and accessing analytics.

2. Dashboard Overview

The Dashboard is your central hub for monitoring all activities in your organization.

Quick Stats

At the top of your dashboard, you'll see four key metrics:

  • Active Projects - Number of projects currently in progress
  • Pending Proposals - Proposals awaiting client approval
  • Open Tickets - Support tickets requiring attention
  • Active Workflows - Automation workflows currently running

Recent Activity

The dashboard displays recent activity across all areas:

  • Recent Projects - Quick access to your latest projects
  • Recent Activity Feed - Timeline of recent actions and updates
  • Pending Proposals - Proposals that need review
  • Open Tickets - Tickets awaiting response
Click on any item in the dashboard to navigate directly to its detail page.

3. Managing Projects

Viewing Projects

The Projects page displays all your automation projects in a sortable, filterable list.

Filtering Options

  • Search - Find projects by name or description (enter at least 3 characters)
  • Status Filter - Filter by: Planned, In Progress, Completed, On Hold, Cancelled
  • Priority Filter - Filter by: Low, Medium, High, Urgent
  • Active Only Toggle - Show only non-completed projects

Creating a New Project

Admin users can create new projects:

  1. Click the "New Project" button in the top right corner
  2. Enter the project name (required)
  3. Add a description of the project scope
  4. Select the priority level
  5. Enter estimated hours for the project
  6. Set start and due dates
  7. Click "Create Project" to save

Project Statuses

Status Description
Planned Project is scheduled but work hasn't started
In Progress Project is actively being worked on
Completed Project has been finished successfully
On Hold Project is temporarily paused
Cancelled Project has been cancelled
Projects with past due dates will be highlighted to indicate they may need attention.

4. Creating Proposals

Understanding Proposals

Proposals are formal documents outlining project scope, deliverables, and pricing for client approval.

Creating a New Proposal

  1. Navigate to the Proposals page and click "New Proposal"
  2. Enter a descriptive title for the proposal
  3. Write the scope of work description
  4. Add line items with descriptions, quantities, and unit prices
  5. Apply any discount percentage if applicable
  6. Set the tax percentage for your region
  7. Set the "Valid Until" date for the proposal
  8. Add client-facing notes and internal notes as needed
  9. Include terms and conditions
  10. Click "Create Proposal" to save

Line Items

Each proposal can have multiple line items. The system automatically calculates:

  • Line item amount (quantity x unit price)
  • Subtotal of all line items
  • Discount amount
  • Tax amount
  • Final total

Proposal Statuses

Status Description
Draft Proposal is being prepared and not yet sent
Sent Proposal has been sent to the client
Accepted Client has approved the proposal
Rejected Client has declined the proposal
Expired Proposal validity date has passed

Revising Proposals

If a proposal needs changes, you can create a revision instead of editing the original. This maintains a history of all versions.

6. Workflow Automation

What are Workflows?

Workflows are automated sequences of actions that streamline your business processes. They can be triggered manually or automatically based on events.

Managing Workflows

  • View Workflows - See all workflow instances with their current status
  • Execute - Manually trigger a workflow to run
  • Pause/Resume - Temporarily stop or restart a workflow
  • View History - See past executions and their results

Workflow Statuses

Status Description
Draft Workflow is being configured
Active Workflow is running and operational
Paused Workflow is temporarily stopped
Error Workflow encountered an issue

Workflow Templates

Browse pre-built templates for common automation scenarios, then customize them for your needs.

7. Support Tickets

Creating a Support Ticket

  1. Navigate to the Tickets page and click "New Ticket"
  2. Enter a clear, descriptive subject line
  3. Provide detailed description of your issue or request (minimum 10 characters)
  4. Select the appropriate category (Technical, Billing, Feature Request, Other)
  5. Set the priority level based on urgency
  6. Click "Submit Ticket" to create

Ticket Priorities

Priority Use When Expected Response
Low General questions or minor issues Within 48 hours
Medium Issues affecting productivity Within 24 hours
High Significant business impact Within 8 hours
Urgent Critical system failures Within 2 hours

SLA Tracking

Each ticket has Service Level Agreement (SLA) tracking:

  • Response SLA - Time until first response
  • Resolution SLA - Time until issue is resolved
  • SLA status is color-coded: green (on track), yellow (at risk), red (breached)

Ticket Statuses

Status Description
Open Ticket has been created and is awaiting attention
In Progress Team is actively working on the issue
Waiting Awaiting response from client
Resolved Issue has been addressed
Closed Ticket is complete and closed

8. Client Onboarding

The Client Onboarding wizard streamlines the process of setting up new projects with proposals.

Step 1: Project Details

Enter the basic project information:

  • Project name (required)
  • Description of the project
  • Priority level
  • Estimated hours
  • Start and due dates

Step 2: Proposal Details

Create the proposal with pricing:

  • Proposal title
  • Scope of work description
  • Line items with pricing
  • Discount and tax settings
  • Validity period

Step 3: Review & Confirm

Review all details before finalizing:

  • Project summary
  • Proposal summary with totals
  • Client-facing and internal notes
  • Terms and conditions
The onboarding wizard creates both the project and proposal in one streamlined flow, saving time and ensuring consistency.

11. FAQ & Troubleshooting

Common Questions

How do I reset my password?

Click "Forgot your password?" on the login page, enter your email, and follow the instructions in the reset email.

Why can't I create a new project?

Only Admin users can create projects. If you need this access, contact your organization administrator.

How do I change my notification preferences?

Go to Settings and update the notification email field. You can also adjust timezone settings there.

What happens when a proposal expires?

Expired proposals cannot be accepted by clients. You can create a revision with a new validity date if needed.

How are SLA times calculated?

SLA times are based on ticket priority. Response SLA measures time to first response, while resolution SLA measures time to close the ticket.

Getting Help

If you encounter issues not covered in this guide:

  1. Create a support ticket through the Tickets page
  2. Include detailed information about the issue
  3. Attach any relevant screenshots if possible
  4. Our support team will respond based on the ticket priority
For urgent issues, set your ticket priority to "Urgent" to receive faster response times.
